## Project Overview

TCC (Trabalho de Conclusão de Curso) at UFU. The goal is to use classical and modern computer vision algorithms to:

1. Estimate the motion vector of the fluid flow (counter-current generated by a tube)
2. Detect and track multiple sperm cells across video frames
3. Predict the future trajectory of each sperm cell given its history and the flow field

Input: microscopy videos containing sperm cells swimming against a tube-generated counter-current.

## Tech Stack

- **Python 3.10+**
- `opencv-python` — video processing, classical optical flow (Lucas-Kanade, Farneback)
- `torch` (PyTorch) — deep learning models (RAFT, LSTM, Transformers)

## Architecture

### Pipeline

The full pipeline is:

```
Video → Frame Extraction → Sperm Detection → Multi-Object Tracking
                        ↘ Optical Flow Estimation ↗
                                                    ↓
                                          Trajectory Prediction
```

### Three Core Modules

**1. Flow Estimation (`src/flow/`)**
- Classical baseline: Farneback dense optical flow (OpenCV)
- Modern baseline: RAFT (Recurrent All-Pairs Field Transforms)
- Output: per-pixel velocity field (u, v) for each frame pair
- The flow field represents the tube-generated counter-current

**2. Tracking (`src/tracking/`)**
- Detection: segmentation-based (thresholding + morphology) or learned detector
- Association: Hungarian algorithm matching detections across frames
- Baselines: SORT (Kalman + Hungarian), DeepSORT, ByteTrack
- Output: trajectories as sequences of (frame_id, sperm_id, x, y)

**3. Prediction (`src/prediction/`)**
- Input features: position history + flow vector at current position
- Classical: Kalman filter (constant velocity model)
- Modern: LSTM/GRU with flow field as auxiliary input
- Output: predicted (x, y) at t+1, t+2, ... t+n

### Key Design Decisions

- Flow field and tracking are **independent modules** — flow can be precomputed and cached
- All experiments are controlled via YAML configs in `configs/` to ensure reproducibility
- Tracking and prediction are evaluated with separate metrics (MOT metrics vs. ADE/FDE)

## Evaluation Metrics

**Tracking:**
- MOTA (Multi-Object Tracking Accuracy)
- MOTP (Multi-Object Tracking Precision)
- HOTA (Higher Order Tracking Accuracy) — primary metric

**Trajectory Prediction:**
- ADE (Average Displacement Error) — mean error over all predicted timesteps
- FDE (Final Displacement Error) — error at the final predicted timestep

## Domain Context

This project relates to **CASA (Computer-Aided Sperm Analysis)**. Key sperm motility metrics:
- VCL (Curvilinear Velocity), VSL (Straight-line Velocity), VAP (Average Path Velocity)
- LIN (Linearity = VSL/VCL)

Sperm cells are small, low-contrast objects with erratic motion, which makes detection and tracking harder than typical MOT benchmarks.

## Algorithms Covered (Classical → Modern)

| Problem | Classical | Modern |
|---|---|---|
| Optical Flow | Lucas-Kanade, Farneback, Horn-Schunck | RAFT, PWC-Net, VideoFlow |
| Detection | Thresholding + Morphology | YOLO, U-Net |
| Tracking | Kalman + Hungarian (SORT) | DeepSORT, ByteTrack, BoT-SORT |
| Prediction | Kalman Filter, Particle Filter | LSTM, GRU, Transformer (TrajNet++) |
